# Suricata Configuration — Elastic Cyber Lab

## Objective

After deploying Suricata, I configured it to operate as a passive Network Intrusion Detection System (NIDS) capable of monitoring attack traffic and exporting structured security telemetry.

The focus was enabling reliable packet capture, structured logging, and compatibility with Elastic SIEM ingestion.

---

## IDS Mode Configuration

Suricata was configured to operate strictly in passive IDS mode.

To avoid interfering with lab traffic:
```text
Inline IPS features were not enabled

NFQUEUE was not configured

No iptables rules were applied

Traffic was inspected without modification
```
This ensured realistic monitoring of attacker behavior.

---

## Interface Monitoring

Suricata was configured to capture traffic from the internal attack network interface.

This allowed inspection of traffic between the attacker and the target systems without monitoring unrelated management traffic.

AF_PACKET was used for high-performance packet capture and multi-threaded processing.

---

## Logging Configuration

To support SIEM integration and analysis, I enabled EVE JSON logging.

This allowed Suricata to export structured security events including:
```text
Intrusion alerts

Network flows

DNS activity

HTTP requests

TLS handshakes

SSH sessions
```
Structured JSON output ensured compatibility with Filebeat parsing and Elasticsearch indexing.

---

## Service Management

After applying configuration changes, I restarted the Suricata service and verified that:

The monitored interface was active

Network traffic was being captured

JSON logs were being written successfully

No packet processing errors were present

Live log monitoring confirmed that Suricata was parsing real attack-network traffic.

---

## Validation

Successful configuration was confirmed when:

Network scan traffic appeared in Suricata logs

Protocol metadata was correctly parsed

Alert events were generated by signature rules

Traffic was captured without packet loss or service disruption

This verified that Suricata was functioning as a reliable network sensor within the SOC lab.

---

## Outcome

Suricata was successfully configured to:

Passively monitor attack traffic

Generate structured security telemetry

Support downstream SIEM ingestion

Provide network visibility for detection engineering

This completed the IDS configuration phase of the lab.
